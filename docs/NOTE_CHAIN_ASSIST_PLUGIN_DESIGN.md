# Note 串辅助放置插件设计文档（细化稿 v0.2）

## 1. 预期行为（请先校对）

以下是实现完成后，用户在编辑器中的预期行为。请你逐条确认是否符合预期。

- 行为 A：插件只处理 `NORMAL` note；`SOUND`/`RAIN` 仅可见但不会被修改。
- 行为 B：插件进入后以“便携浮动按钮”作为主入口，固定按钮只保留一个兜底入口。
- 行为 C：用户可在主编辑区叠加层直接拖拽锚点/手柄，实时看到 note 串预览。
- 行为 C1：预览与辅助显示元素可独立开关（如控制点、手柄、采样点、参考线、标签文本）。
- 行为 D：高级设置不挤占主界面，统一在浮动窗口/弹窗中完成。
- 行为 E：支持多窗口协作，窗口可停靠、标签合并、拆分恢复。
- 行为 F：颜色编辑不是自由 RGB，而是“分母序列编排并应用”。
- 行为 G：任何应用操作保持谱面格式不变，不向 `.mc` 注入新字段。
- 行为 H：可保存“曲线工程文件”（与唯一谱面一对一关联），用于继续编辑。
- 行为 I：可导入/导出“样式文件”，样式不绑定谱面，可跨谱面复用。
- 行为 J：提交时通过单次 batch edit 进入一个 undo 步骤，便于整体撤销。
- 行为 K：预览态不落盘，只有“提交”才写入谱面。
- 行为 L：编辑操作本质仍在主程序编辑区完成；插件以“可切换工具”方式提供增强能力。
- 行为 M：当切换到该工具状态时，启用插件增强交互；切回其他工具后恢复宿主默认行为。

校对建议：

- 若以上某条不符合预期，请直接指出“行为字母 + 你希望的改法”。

---

## 2. 范围与边界

本插件用于 Malody Catch 的 note 串构建、曲线变形和颜色序列应用，强调快速迭代与可重复编辑。

明确范围：

- 曲线驱动批量生成/调整 `NORMAL` note。
- 基于分母序列的颜色节奏设计与应用。
- 浮动工作台（按钮、面板、弹窗）协同。
- 外部 sidecar 数据管理（曲线工程 + 样式）。

不在范围：

- 修改 `.mc` 格式定义。
- 自定义 RGB/HSV 颜色写回谱面。
- 替代编辑器原有点放、框选、移动等基础能力。
- 修改 SOUND/RAIN note 数据结构。

## 3. 交互与 UI 策略

## 3.1 入口优先级

- 工具形态：作为主程序中的“可切换工具”集成，而非独立编辑器。
- 第一入口：便携浮动按钮（可拖动、可折叠、可吸附边缘）。
- 第二入口：插件工作台浮动窗口（用于复杂参数编辑）。
- 固定入口：仅一个主菜单项或工具栏兜底按钮（可开关）。

## 3.2 工作台窗口模型

窗口角色：

- `Primary`：曲线参数窗口（锚点、采样、约束）。
- `Secondary`：颜色序列窗口（分母序列、编组、应用策略）。
- `Library`：模板与样式库窗口（导入导出、收藏、预设）。

窗口行为：

- 支持浮动、停靠、Tab 合并。
- 支持布局预设：`Compact`、`Dual`、`Advanced`。
- 支持布局保存/恢复（按用户本地配置）。

## 3.3 主编辑区直操行为

- 锚点左键拖拽：移动锚点。
- 手柄拖拽：调整切线方向与长度。
- 双击锚点：角点/平滑点切换。
- `Esc`：取消当前编辑手势。
- `Enter`：提交当前预览为一次批量编辑。
- 显示开关：可随时开关 `预览串`、`控制点`、`手柄`、`采样点`、`辅助标注`。

说明：插件工具态开启时可拦截输入并增强主编辑区；关闭时恢复宿主默认编辑手势。

## 4. 数据与持久化设计

## 4.1 曲线工程文件（谱面绑定）

用途：保存可持续编辑状态。

建议内容：

- 曲线锚点、手柄、模板参数、采样参数。
- 当前工作台状态（选中链、显示选项、局部开关）。
- 引用样式 ID（可选）。

关联规则：

- 一份谱面对应一份曲线工程文件（1:1）。
- 关联键使用：`chart_path_canonical + chart_fingerprint`。
- 若路径变化，优先指纹恢复，失败则提示手动重绑定。

## 4.2 样式文件（谱面无关）

用途：跨谱面复用视觉与编排方案。

建议内容：

- 分母序列模板。
- 曲线模板参数预设。
- 面板布局偏好（可选）。

导入导出：

- 支持单文件导入/导出。
- 支持批量导入样式库目录。
- 导入时按 `style_id` 处理重名（覆盖/并存/重命名）。

## 4.3 后缀策略（待定）

当前保留占位：

- 曲线工程：`*.{curve_ext_tbd}`
- 样式文件：`*.{style_ext_tbd}`

在首次实现时统一冻结后缀并回填文档。

## 5. 功能细化

## 5.1 曲线与采样

- 曲线类型：Bezier、平滑折线、模板变形曲线。
- 采样模式：
- 弧长均匀采样（视觉均匀）。
- 节拍步进采样（节奏一致）。
- 约束模式：
- 保持 beat，仅改 x。
- 保持 x 分布，仅改 beat。
- beat + x 同步变形。

## 5.2 颜色序列（分母驱动）

- 序列定义：例如 `[4,8,12,16]`。
- 推进模式：循环、分组推进、反转、镜像。
- 应用对象：仅 `NORMAL` note。
- 精度规则：仅允许精确转换；不精确时按策略“跳过/回退到安全分母”。

## 5.3 提交模型

- 预览：仅内存态，不写谱面。
- 提交：通过 `buildBatchEdit` 一次性提交。
- Undo：提交后在宿主表现为一个 undo 步骤。

## 7. 可行性评估

可立即落地：

- 浮动按钮 + 浮动面板 + 样式导入导出。
- 非直操的模板生成与批量应用。

需宿主改造后落地：

- 画布锚点直操（`canvas_interaction`）。
- 工作台停靠合并（`panel_workspace`）。

高风险点：

- 进程插件高频事件延迟。
- 大规模预览刷新造成卡顿。

建议：交互核心优先 Native 插件；进程插件侧重离线批处理。

## 8. 分阶段实施

## Phase 1（可用版本）

- 便携浮动按钮 + 单面板参数编辑。
- 模板生成 + 预览 overlay + 单步提交。
- 样式文件导入导出。

验收：

- 可生成并提交 note 串。
- 可导入导出样式并跨谱复用。

## Phase 2（直操版本）

- 引入 `canvas_interaction`。
- 支持锚点/手柄直接拖拽编辑。
- 支持多窗口停靠合并。

验收：

- 直操响应流畅。
- 与原生编辑手势切换明确、无冲突。

## Phase 3（高级版本）

- 自相似模板扩展。
- 布局预设保存/恢复/共享。
- 工程文件关联恢复能力完善。

---

## 9. 风险与对策

- 风险：输入事件拥塞导致卡顿。
- 对策：事件节流、增量 overlay、超时降级。

- 风险：路径变化导致曲线工程文件失联。
- 对策：路径 + 指纹双因子关联，提供重绑定向导。

- 风险：分母转换导致时间漂移。
- 对策：严格精确转换检查，失败即跳过或提示用户。
